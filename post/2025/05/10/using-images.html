Using Images in Git Pages React Framework

This post demonstrates how to include images in your markdown posts using the Git Pages React Framework.

Basic Image Syntax

The basic syntax for including an image in markdown is:

![Alt text](/path/to/image.jpg){: loading="lazy" }

Where:

  • Alt text is the alternative text for the image (for accessibility)
  • path/to/image.jpg is the path to your image file

Image Path Options

The Git Pages React Framework supports several ways to reference images:

1. Using images from the assets/images directory

The recommended way is to place your images in the assets/images directory and reference them directly:


<picture>
  <source type="image/webp" srcset="/assets/webp/logo-320.webp 320w, /assets/webp/logo-640.webp 640w, /assets/webp/logo-960.webp 960w, /assets/webp/logo-1280.webp 1280w" sizes="(max-width: 1280px) 100vw, 1280px">
  <source type="image/png" srcset="/assets/responsive/logo-320.png 320w, /assets/responsive/logo-640.png 640w, /assets/responsive/logo-960.png 960w, /assets/responsive/logo-1280.png 1280w" sizes="(max-width: 1280px) 100vw, 1280px">
  <img src="/assets/images/logo.png" alt="Git Pages React Logo" loading="lazy">
</picture>

This will automatically look for the image in the assets/images directory.

2. Using a relative path

You can also use a relative path to the assets directory:


<picture>
  <source type="image/webp" srcset="/assets/webp/logo-320.webp 320w, /assets/webp/logo-640.webp 640w, /assets/webp/logo-960.webp 960w, /assets/webp/logo-1280.webp 1280w" sizes="(max-width: 1280px) 100vw, 1280px">
  <source type="image/png" srcset="/assets/responsive/logo-320.png 320w, /assets/responsive/logo-640.png 640w, /assets/responsive/logo-960.png 960w, /assets/responsive/logo-1280.png 1280w" sizes="(max-width: 1280px) 100vw, 1280px">
  <img src="/assets/images/logo.png" alt="Git Pages React Logo" loading="lazy">
</picture>

3. Using external images

You can use images from external sources by providing the full URL:

![External Image](https://example.com/images/sample.jpg){: loading="lazy" }

Image Styling

Images are automatically styled with:

  • Responsive width (max-width: 100%)
  • Proper spacing
  • Centered alignment

Best Practices

  1. Always include alt text - This is important for accessibility
  2. Optimize your images - Compress images before adding them to your project
  3. Use appropriate file formats:
    • JPEG for photographs
    • PNG for graphics with transparency
    • SVG for vector graphics
    • WebP for better compression (with fallbacks)

Conclusion

Adding images to your posts enhances the visual appeal and helps convey information more effectively. The Git Pages React Framework makes it easy to include and manage images in your markdown content.
